 Class Methods|कक्षा/वर्ग की विधियाँ:

We learned from the Methods topic that methods are declared within a class, and that they are used to perform certain actions: हमने "विधियाँ" मे से सीखा है कि एक वर्ग(class) के भीतर विधियाँ घोषित किए जाते हैं, और उनका उपयोग कुछ क्रियाओं को करने के लिए किया जाता है:

Example:

Create a विधि(method) named "मेरीविधि()" in "मुख्य" :

उदाहरण:

"मुख्य" में "मेरीविधि ()" नाम की एक विधि बनाएँ:
कक्षा मुख्य { स्थिर रिक्त मेरीविधि(){ तंत्र.बाहर.प्रिंट("नमस्ते !!"); } }

मेरीविधि() prints a text (the action), when it is called. To call a विधि(method), write the method's name followed by two parentheses () and a semicolon ; हमने "विधियाँ" मे से सीखा है कि एक वर्ग(class) के भीतर विधियाँ घोषित किए जाते हैं, और उनका उपयोग कुछ क्रियाओं को करने के लिए किया जाता है:

Example:

Inside "मुख्य", call "मेरीविधि()":

उदाहरण:

"मुख्य" में "मेरीविधि()" विधि बुलाए/चलाए :
कक्षा मुख्य { स्थिर रिक्त मेरीविधि(){ तंत्र.बाहर.प्रिंट("नमस्ते !!"); } सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ मेरीविधि(); } }//नमस्ते !! लिख देगा



 Static vs. Public|स्थिर बनाम सार्वजनिक

You have seen some programs that have either स्थिर(static) or सार्वजनिक(public) attributes(विशेषता/गुण) and methods(विधियाँ).
In the below example, we created a स्थिर(static) method, which means that it can be accessed without creating an object(वस्तु) of the class(वर्ग), unlike सार्वजनिक(public), which can only be accessed by object(वस्तु): आपने कुछ ऐसे प्रोग्राम देखेंगे जिनमें या तो स्थिर(static) या सार्वजनिक(public) विशेषताएँ(attributes) और विधियाँ(methods) होती हैं।
नीचे दिए उदाहरण में, हमने एक स्थिर(static) विधि बनाई, जिसका अर्थ है कि सार्वजनिक(सार्वजनिक) के विपरीत वर्ग(class) का एक वस्तु(object) बनाए बिना इसेका इस्तेमाल (एक्सेस) किया जा सकता है , जबकि सार्वजनिक केवल वस्तु(object) द्वारा ही इस्तेमाल(एक्सेस) किया जा सकता है:

Example:

An example to demonstrate the differences between स्थिर(static) and सार्वजनिक(public) methords :

उदाहरण:

स्थिर(स्थिर) और सार्वजनिक(सार्वजनिक) विधियों के बीच अंतर स्पष्ट करने के लिए एक उदाहरण:
कक्षा मुख्य { स्थिर रिक्त मेरीपहलीविधि(){ तंत्र.बाहर.प्रिंट("स्थिर विधियाँ बिना वर्ग की वस्तु बनाए इस्तेमाल की जा सकती है|"); } सार्वजनिक रिक्त मेरीदुसरीविधि(){ तंत्र.बाहर.प्रिंट("सार्वजनिक विधियों को वर्ग की वस्तु बना कर ही इस्तेमाल किया जाना चाहिए|"); } सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ मेरीपहलीविधि(); //यहाँ वस्तु की आवश्यकता नहीं है मुख्य मेरीवस्तु =नई मुख्य(); //यहाँ वस्तु की आवश्यकता है मेरीवस्तु.मेरीदुसरीविधि(); } }

We will learn more about these keywords (called modifiers) in the Modifiers. हम संशोधक में इन शब्दों (जिन्हें संशोधक कहा जाता है) के बारे में अधिक जानेंगे।



 Access Methods With an Object| वस्तु से विधियों तक पहुँचे

Example:

Create a "कार" object named myCar. Call the fullThrottle() and speed() methods on the myCar object, and run the program:

उदाहरण:


वर्ग कार { //तेज़चले() विधि बनाए सार्वजनिक रिक्त तेज़चले() { तंत्र.बाहर.प्रिंट("कार जितनी तेज जा सकती है उतनी तेज चल रही है ! "); } //गति() विधि बनाए और उसमे अधिकतमगती मानदंड/parameter बनाए सार्वजनिक रिक्त गति(अंक अधिकतमगती){ तंत्र.बाहर.प्रिंट("अधिकतम गति है :" + अधिकतमगती); } //गति() विधि बनाए और उसमे अधिकतमगती मानदंड/parameter बनाए सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ कार मेरिकार = नई कार(); मेरिकार.तेज़चले(); मेरिकार.गति(150); } }

    Example explained
  • 1) We created a custom "कार" class with the वर्ग keyword.
    (you can also use कक्षा here as they both are the same )
  • 2) We created the "तेज़चले()" and "गति()" विधियाँ(methods) in the "कार" वर्ग(class).
  • 3) The "तेज़चले()" विधि(method) and the "गति()" विधि(method) will print out some text, when they are called.
  • 4) The "गति()" विधि(method) accepts an अंक parameter called "अधिकतमगती" - we will use this in 8).
  • 5) In order to use the "कार" class and its methods(विधियाँ), we need to create an object(वस्तु) of the "कार" class.
  • 6) Then, go to the जरूरी() method, which you know by now is a built-in विधि(method) that runs your program (any code inside जरूरी() is executed).
  • 7) By using the नई keyword we created a वस्तु(object) with the name "मेरिकार".
  • 8) Then, we call the "तेज़चले()" and "गति()" methods(विधियाँ) on the "मेरिकार" object/वस्तु , and run the program using the name of the वस्तु(object) i.e."मेरिकार" , followed by a dot (.), followed by the name of the method (तेज़चले(); and गति(150); ). Notice that we add an अंकparameter of 150 inside the गति() method(विधि).
    उदाहरण समझे
  • 1) हमने वर्ग शब्द के साथ एक अलग नया "कार" वर्ग बनाया। (आप यहां कक्षा का भी प्रयोग कर सकते हैं क्योंकि जैसा हम जानते है दोनों एक ही हैं)
  • 2) हमने "कार" वर्ग(class) में "तेज़चले()" और "गति()" विधियाँ(methods) बनाई हैं।
  • 3) "तेज़चले ()" विधि(method) और "गति()" विधि (methord) कुछ प्रिंट करेंगे, जब उन्हें बुलाया(इस्तेमाल मे लाया) जाएगा।
  • 4) "गति ()" विधि (विधि) एक अंक मानदंड/पैरामीटर को स्वीकार करती है जिसे "अधिकतमगती" कहा जाता है/गया है - हम इसका उपयोग 8) में करेंगे।
  • 5) "कार" वर्ग और उसकी विधियों (methords) का उपयोग करने के लिए, हमें "कार" वर्ग की एक वस्तु(object) बनाने की आवश्यकता है।
  • 6) फिर , जरूरी() विधि(methord) पर जाएं, जिसे अब तक आप जानते हैं कि यह एक built-in विधि(methord) है जो हमारे प्रोग्राम को चलता है |(जरूरी() के अंदर का कोड ही सबसे पहले निष्पादित/रन होता है)
  • 7) नई शब्द का उपयोग करके हमने "मेरिकार" नाम से एक वस्तु(object) बनाया है।
  • 8) फिर, हम "तेज़चले ()" और "गति ()" विधियों को "मेरिकार" वस्तु(object) पर बुलाते हैं या कहे इस्तेमाल करते है (या कॉल करते है)|और फिर प्रोग्राम को वस्तु के नाम("मेरिकार") , फिर एक बिंदु (.), उसके बाद विधि का नाम (ते ज़चले (); तथा गति(150); ) लिख कर चलाते है.ध्यान दें कि हम गति () विधि (methord) के अंदर अंक 150 का पैरामीटर जोड़ते हैं।

Remember that..

The dot (.) is used to access the object's(वस्तु के) attributes(गुण/विशेषता) and methods(विधियाँ).
To run(call) a method(विधि) , write the method name(विधि का नाम) followed by a set of parentheses (), followed by a semicolon ;.

याद रखे..

बिंदु (.) का प्रयोग वस्तु(object's) के गुण/विशेषता (attributes) और विधियों (methords) तक पहुँचने के लिए किया जाता है।
एक विधि(methord) चलाने (कॉल करने) के लिए, विधि का नाम(methord name) लिखें और उसके बाद कोष्ठकों का एक समूह (), उसके बाद अर्धविराम ; लिखें।।

We have seen some classes with access modifier सार्वजनिक and other classes without it(आमवर्ग (default classes) . The difference between the two is explained in detail in modifiers topic,as सार्वजनिक and आम are modifiers.They are amongst the most important topics . You can check them there. हमने संशोधक सार्वजनिक के साथ कुछ कक्षाएं और इसके बिना भी अन्य कक्षाएं देखी हैं (आमवर्ग (default classes)। यह एक अहम विषय है |दोनों को संशोधक(modifiers) में विस्तार से समझाया गया है, (क्योंकि सार्वजनिक और आम संशोधक हैं) आप उन्हें वहां पढ़ सकते हैं।
