 Inner Classes| भीतरी वर्ग |अंतरिम वर्ग | अंदर के वर्ग:

It is also possible to nest वर्ग(classes) (a वर्ग(class) within a वर्ग(class)). The purpose of nested वर्ग(classes) is to group वर्ग(classes) that belong together, which makes your code more readable and maintainable.To access the भीतरी वर्ग(inner class), create an वस्तु(object) of the outer वर्ग(class), and then create an वस्तु(object) of the भीतरी वर्ग(inner class): किसी एक वर्ग(class) को दुसरे वर्ग(class) के भीतर बनाना भी संभव है)। इसका (भीतरी वर्ग(inner class))का उद्देश्य वर्ग(class) को एक साथ एकत्रित करना है, जो आपके संहिता/कोड को अधिक पठनीय और अच्छा बनाए रखने मे मदद करता है। इसका इस्तेमाल करने के लिए बाहरी वर्ग(outer class) की एक वस्तु(object) बनाएँ और फिर भीतरी वर्ग(inner class) की एक वस्तु(object) बनाएँ:

Example:

उदाहरण:
वर्ग बाहार {//बाहरी वर्ग outer class अंक अ = 100 ; वर्ग अंदर {//भितरी वर्ग inner class अंक ब =200 ; } } वर्ग मुख्य{ सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ बाहार मेराबाहार =नई बाहार();//मेरा बाहरी वर्ग बाहार.अंदर मेराअंदर =मेराबाहार.नई अंदर(); तंत्र.बाहर.प्रिंट(मेराबाहार.अ + मेराअंदर.ब ); } }//100 +200=300 लिख देगा



 Private Inner Class|निजीभितरी वर्ग |निजीअंदर के वर्ग

Unlike a "regular" वर्ग(class), an भितरी वर्ग(inner class) can be निजी(private) or सुरक्षित(protected). If you don't want outside वस्तु(objects) to access the भितरी वर्ग(inner class), declare the class as निजी(private): "सामान्य" वर्ग के विपरीत, एक भितरी वर्गनिजी(private) या सुरक्षित(protected) हो सकता है। यदि आप नहीं चाहते कि बाहरी वस्तुएँ भीतरी कक्षा/वर्ग तक पहुँचें, तो कक्षा/वर्ग को निजी के रूप में घोषित करें:

Example:

उदाहरण:
वर्ग बाहार {//बाहरी वर्ग outer class अंक अ = 100 ; निजी वर्ग अंदर {//भितरी वर्ग inner class अंक ब =200 ; } } वर्ग मुख्य{ सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ बाहार मेराबाहार =नई बाहार();//मेरा बाहरी वर्ग बाहार.अंदर मेराअंदर =मेराबाहार.नई अंदर(); //उपरोक्त कथन एक त्रुटि देगा क्योंकि अंदर निजी है तंत्र.बाहर.प्रिंट(मेराबाहार.अ + मेराअंदर.ब ); } }//एक त्रुटि देगा क्योंकि अंदर निजी है

 Static Inner Class|स्थिरभितरी वर्ग

An भितरी वर्ग(inner class) can also be स्थिर(static), which means that you can access it without creating an वस्तु(object) of the बाहरी वर्ग (outer class): एक भितरी वर्ग(inner class) स्थिर(static) भी हो सकता है, जिसका अर्थ है कि आप स्थिर(static) का वस्तु (object) बनाए बिना उस तक पहुंच सकते हैं:

Example:

उदाहरण:


वर्ग बाहार {//बाहरी वर्ग outer class अंक अ = 100 ; स्थिर वर्ग अंदर {//भितरी वर्ग inner class अंक ब =200 ; } } वर्ग मुख्य{ सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ बाहार.अंदर मेराअंदर =नई बाहार.अंदर(); तंत्र.बाहर.प्रिंट(मेराअंदर.ब ); } }//200 लिख देगा

Note :

Just like स्थिर(static) attributes and methods, a स्थिर भितरी वर्ग(static inner class) does not have access to members of the बाहरी वर्ग (outer class).

ध्यान दे:

स्थिर(static) विशेषताओं और विधियों की तरह, स्थिर भितरी वर्ग static inner class) के पास बाहरी वर्ग (outer class)के सदस्यों तक पहुंच नहीं है।

 Access Outer Class From Inner Class|भितरी वर्ग से बाहरी वर्ग तक पहुचे

One advantage of भितरी वर्ग(inner classes), is that they can access गुण(attributes) and विधियाँ(methods) of the बाहरी वर्ग(outer class): भितरी वर्ग(inner classes) का एक फायदा यह है कि वे बाहरी वर्ग(outer class) के गुण(attributes) और विधियाँ(methods) तक पहुँच सकते हैं:

Example:

उदाहरण:
वर्ग बाहार {//बाहरी वर्ग outer class अंक अ = 100 ; वर्ग अंदर {//भितरी वर्ग inner class सार्वजनिक अंक मेरीविधि() { वापस अ ; } } } वर्ग मुख्य{ सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ बाहार मेराबाहार = नई बाहार(); बाहार.अंदर मेराअंदर = मेराबाहार.नई अंदर(); तंत्र.बाहर.प्रिंट(मेराअंदर.मेरीविधि()); } }//100 लिख देगा
