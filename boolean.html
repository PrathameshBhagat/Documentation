Boolean | दोहरा डेटा प्रकार
 →NEXT

Very often, in programming, we will need a data type that can only have one of two values, like:
  • YES / NO
  • ON / OFF
  • TRUE / FALSE
For this, we have a दोहरा(boolean) data type, which can store सचtrue or झूठfalse values. कई बार, प्रोग्रामिंग में, हमें एक डेटा प्रकार की आवश्यकता होगी जिसमें केवल दो में से एक मान हो, जैसे:
  • हा / ना
  • शुरू / बंद
  • सच / झूठ
इसके लिए हमारे पास एक दोहरा डेटा प्रकार है, जो सच/सही या झूठ/गलत मान संग्रहीत कर सकता है।

 Boolean Values|दोहरा के मान

A दोहरा(boolean) type is declared with the दोहरा(boolean) keyword and can only take the values सच(true) or झूठ(false): एक दोहरा(boolean) प्रकार दोहरा(boolean) शब्द(कीवर्ड) के साथ घोषित किया जाता है और केवल सच(true) या झूठा(false) मान ले सकता है :

Example:

उदाहरण:
दोहरा प्रोग्रामिंग_आसान_है = सच ; दोहरा बिल्ली_भोंक_सकती_है = झूठ; तंत्र.बाहर.प्रिंट(प्रोग्रामिंग_आसान_है);//sach तंत्र.बाहर.प्रिंट(बिल्ली_भोंक_सकती_है);//झूठ


But, it is more common to return दोहरा boolean values from boolean expressions, for conditional testing (see below). लेकिन, अधिक सामान्य है दोहरी अभिव्यक्तियो(boolean expressions ) से दोहरा(boolean) मान लौटाना ।(नीचे देखे )


 Boolean Expression| दोहरी अभिव्यक्ति | बूलियन अभिव्यक्तियाँ

A दोहरी अभिव्यक्ति(Boolean expression) returns a दोहरा(boolean) value: सच(true) or झूठ(false) .

This is useful to build logic, and find answers.

For example, you can use a comparison operator, such as the greater than > (>)operator, to find out if an expression (or a variable) is सच(true) or झूठ(false): एक दोहरी अभिव्यक्ति(Boolean expression) एक दोहरा(boolean) मान देता है: सच(true) या झूठ(false) ।

यह तर्क बनाने और उत्तर खोजने के लिए उपयोगी है।

उदाहरण के लिए, आप एक तुलना संचालक(operator) का उपयोग कर सकते हैं, जैसे > (>)संचालक(operator) से अधिक, खोजने के लिए यदि कोई व्यंजक (या एक चर(variable)) सच(true) या झूठ/गलत(false) है:

Example:

उदाहरण:
अंक अ = 100; अंक ब = 90; तंत्र.बाहर.प्रिंट( अ > ब );//सच है क्योंकि 100 , 90 से ज्यादा है



Example:

Or even easier:

उदाहरण:

या इससे भी आसान:
तंत्र.बाहर.प्रिंट( 100 > 90 );//सच है क्योंकि 100 , 90 से ज्यादा है

In the below example, we use the equal to == संचालक(operator) to evaluate an expression: नीचे दिए गए उदाहरण में, हम एक व्यंजक(expression) का मूल्यांकन(evaluation) करने के लिए के बराबर संचालक(operator)==का उपयोग करते हैं:

Example:

Or even easier:

उदाहरण:

या इससे भी आसान:
अंक अ= 100 ; तंत्र.बाहर.प्रिंट( अ == 100 );//सच है क्योंकि अ का मूल्य 100

Example:

उदाहरण:
तंत्र.बाहर.प्रिंट( 100 == 90 );//झूठ है क्योंकि 100 , 90 के बराबर नहीं है



 Real Life Example|वास्तविक जीवन उदाहरण

Let's think of a "real life example" where we need to find if a person is old enough to vote.

In the example given below, we use the >=तुलना संचालक ( comparison operator) to find out if the "उम्र"(25) is ज्यादा या अरबर है (greater than OR equal) to the voting age limit, which is set to 18: आइए एक "वास्तविक जीवन उदाहरण" के बारे में सोचें जहां हमें यह पता लगाने की आवश्यकता है की क्या कोई व्यक्ति मतदान करने के लिए पर्याप्त आयु का है या नहीं ।

नीचे दिए गए उदाहरण में, हम >= तुलना संचालक (comparison operator) का उपयोग करते हैं पता करने के लिए की क्या "उम्र"(25) अधिक या बराबर है(greater than OR equal) मतदान की आयु सीमा के लिए, जो कि 18 पर स्थापित है, या नहीं |

Example:

उदाहरण:
अंक मेरीउम्र = 25 ; अंक मतदानआयु = 18 ; तंत्र.बाहर.प्रिंट( मेरीउम्र >= मतदानआयु );

An even better approach (as we are on a roll now), would be to wrap the code above in an अगर....या(if...else) statement, so we can perform different actions depending on the result: एक और भी बेहतर तरीका उपरोक्त संहिता/कोड को अगर....या(if...else) कथन में लिखना होगा, ताकि हम अलग-अलग परिणाम के आधार पर कार्रवाई/प्रदर्शन कर सकें :

Example:

Output "आप मतदान कर सकते है!" if "मेरीउम्र" is greater than or equal to 18. Otherwise output "आप मतदान नहीं कर सकते है":

उदाहरण:
अंक मेरीउम्र = 25 ; अंक मतदानआयु = 18 ; अगर (मेरीउम्र >= मतदानआयु){ तंत्र.बाहर.प्रिंट("आप मतदान कर सकते है!"); तंत्र.बाहर.प्रिंट(" आपकी उम्र पर्याप्त है !"); } या { तंत्र.बाहर.प्रिंट(" आप मतदान नहीं कर सकते है,"); तंत्र.बाहर.प्रिंट(" आपकी उम्र पर्याप्त नहीं है !"); }

Note :
