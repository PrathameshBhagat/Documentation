 Interface | अंतरापृष्ठ :

Another way to achieve abstraction is with अंतरापृष्ठ(interfaces).

An अंतरापृष्ठ(interfaces) is a completely "सार वर्ग" ("abstract class") that is used to group related विधियाँ(methods) with empty bodies: सार निकालना/अमूर्तता( abstraction) प्राप्त करने का एक अन्य तरीका अंतरापृष्ठ(interfaces) है। एक अंतरापृष्ठ(interfaces) एक पूरी तरह से "सार वर्ग" ("abstract class") है जिसका उपयोग सार विधियों (abstract methods)(खाली विधियों(methods))को एकत्रित करने के लिए किया जाता है:

Example:

उदाहरण:
अंतरापृष्ठ जानवर { //अंतरापृष्ठ की विधियो मे शरीर नहीं होता है सार्वजनिक रिक्त जानवरकिआवाज़(); //क्योंकि अंतरापृष्ठ की सभी विधियाँ सार विधि होती है सार्वजनिक रिक्त भागों(); }


To access the अंतरापृष्ठविधियाँ (interface methods), the अंतरापृष्ठ(interface) must be "implemented" (kinda like विरासेगा/inherited) by another वर्ग(class) with the बसाये keyword (instead of विरासेगा(extends)). The body of the interface method is provided by the "implement" class: साधे अंतरापृष्ठविधियाँ(interface methods) तक पहुँचने के लिए, अंतरापृष्ठ(interface) को एक अन्य वर्ग(class) द्वारा "कार्यान्वित" (विरासीत की तरह) होना चाहिए। बसये कीवर्ड (विरासेगा(बढ़ता है) के बजाय)। इंटरफ़ेस विधि का मुख्य भाग "कार्यान्वयन" वर्ग द्वारा प्रदान किया जाता है:

Example:

उदाहरण:


//अंतरापृष्ठ Interface अंतरापृष्ठ जानवर { सार्वजनिक रिक्त जानवरकीआवाज़();//अंतरापृष्ठ कीविधियो मे शरीर नहीं होता है सार्वजनिक रिक्त सोजाओ();//क्योंकी अंतरापृष्ठ की सभी विधियाँ सार विधि होती है } // Dog लागू("implements)" the Animal interface //कुत्ता वर्ग मे जानवर अंतरापृष्ठ का इस्तेमाल वर्ग कुत्ता लागू जानवर { सार्वजनिक रिक्त जानवरकीआवाज़(){ //जानवरकीआवाज़ का शरीर/कोड यहाँ दीया गया है तंत्र.बाहर.प्रिंट("कुत्ता भोंकेगा | भों भों .. | "); } सार्वजनिक रिक्त सोजाओ(){ //भागो का शरीर यहाँ दिया गया है तंत्र.बाहर.प्रिंट("कुत्ता सो गया | Zzz... "); } } वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ कुत्ता मेराकुत्ता = नई कुत्ता();//कुत्ता वर्ग की नई वस्तु मेराकुत्ता.जानवरकीआवाज़(); मेराकुत्ता.सोजाओ(); } }

Note :

  • Like सार वर्ग (abstract classes),अंतरापृष्ठ (interfaces) cannot be used to create वस्तु (objects) (in the above example, it is not possible to create an "जानवर" वस्तु(object) in मुख्य)
  • अंतरापृष्ठ विधियाँ(Interface methods) do not have a body - the body is provided by the "implement" वर्ग(class) through अधिरोहना (overriding) it .
  • On implementation of an अंतरापृष्ठ (interface), you must अधिरोहे (override) all of its विधियाँ(methods) if not an error/exception will be thrown .
  • अंतरापृष्ठ की विधियाँ(Interface methods) are by default सार(abstract) and सार्वजनिक(public)
  • अंतरापृष्ठ के गुण/विशेषताएं(Interface attributes/variables) are by default सार्वजनिक(public),स्थिर(static) and अंतिम(final)
  • An अंतरापृष्ठ(interface) cannot contain a निर्मातक विधियाँ (constructor) (as it cannot be used to create वस्तु(objects))

Why And When To Use अंतरापृष्ठ(Interfaces)? :

  1. 1)Hide certain not very important details and only show the important details of an वस्तु(object)(i.e. interface).
  2. 2) Our programming language does not support "multiple inheritance" (a वर्ग(class) can only inherit from one उच्चवर्ग(superclass)). However, it can be achieved with अंतरापृष्ठ(interfaces), because the वर्ग(class) can implement multiple अंतरापृष्ठ(interfaces). Note: To implement multiple अंतरापृष्ठ(interfaces), separate them with a comma , (see example below).

ध्यान दे:

  • जैसे सार वर्ग(abstract classes) , अंतरापृष्ठ (interfaces) का भी उपयोग वस्तु (object)बनाने के लिए नहीं किया जा सकता है (उपरोक्त उदाहरण में, मुख्य मे "जानवर" की वस्तु (object) बनाना संभव नहीं है)
  • अंतरापृष्ठ विधियाँ(Interface methods) में शरीर/कोड नहीं होता है - उसे शरीर/कोड, लागू ("implement") करने वाला वर्ग(class) प्रदान करता है।
  • अंतरापृष्ठ (interface) का इस्तेमाल करने पर, आपको इसकी सभी विधियाँ (methords) को अधिरोहीत(override) करना होगा(याद करे हमने बहुरूपता मे अधिरोहण (override) देखा था |)
  • अंतरापृष्ठ की विधियाँ(Interface methods) मूल रूप से ही(पहले से ही) सार(abstract) और सार्वजनिक(public) होती है |
  • अंतरापृष्ठ के गुण/विशेषताएं(Interface attributes/variables) मूल रूप से ही (पहले से ही)सार्वजनिक(public),स्थिर(static) और अंतिम(final) होती है |
  • किसी अंतरापृष्ठ(interface) मे निर्मातक/रचनाकार विधियाँ (constructor) शामिल नहीं हो सकतीं (क्योंकि इसका उपयोग वस्तु (objects) बनाने के लिए नहीं किया जा सकता)

अंतरपृष्ठ (interface) का उपयोग क्यों और कब करें? :

  1. कुछ गैर-महत्वपूर्ण विवरण छिपाने और केवल एक वस्तु(object) (i.e. interface) के महत्वपूर्ण विवरण दिखाएं।
  2. हमारी प्रोग्रामिंग भाषा "मल्टीपल इनहेरिटेंस" का समर्थन नहीं करती है। (एक वर्ग (class) केवल एक उच्चवर्ग (superclass) से प्राप्त हो सकता है)। हालाँकि, यह अंतरापृष्ठ (Interface) के साथ प्राप्त किया जा सकता है, क्योंकि वर्ग (class) कई अंतरापृष्ठ (Interface) लागू कर सकता है। टिप्पणी: एकाधिक अंतरापृष्ठ (Interface) लागू करने के लिए, उन्हें अल्पविराम , से अलग करें (नीचे उदाहरण देखें)।



 Multiple Interfaces|एकाधिक अंतरापृष्ठ

To use multiple अंतरापृष्ठ(interfaces), separate them with a comma , : एकाधिक अंतरापृष्ठ(interfaces) का उपयोग करने के लिए, उन्हें अल्पविराम (comma ,) से अलग करें , :

Example:

उदाहरण:


अंतरापृष्ठ कुत्ता { सार्वजनिक रिक्त बहुततेज़दौड़ेगा(); } अंतरापृष्ठ बिल्ली { सार्वजनिक रिक्त बहुततेज़दीवारचढ़ेगी(); } वर्ग पालतूजानवर लागू कुत्ता ,बिल्ली { सार्वजनिक रिक्त बहुततेज़दौड़ेगा(){ तंत्र.बाहर.प्रिंट("कुत्ता तेज़ दौड़ते हुए चला गाया | "); } सार्वजनिक रिक्त बहुततेज़दीवारचढ़ेगी(){ //भागो का शरीर यहाँ दिया गया है तंत्र.बाहर.प्रिंट(" बिल्ली दीवार पर चढ़ गयी | "); } } वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ पालतूजानवर मेराजानवर = नई पालतूजानवर();//कुत्ता वर्ग की नई वस्तु मेराजानवर.बहुततेज़दौड़ेगा(); मेराजानवर.बहुततेज़दीवारचढ़ेगी(); } }
