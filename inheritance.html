 Inheritance (Subclass and Superclass)|विरासत (उपवर्ग और उच्चवर्ग):

In programming , it is possible to inherit गुण(attributes) and विधि(methods) from one वर्ग)class) to another. We group the "विरासत विषय"("inheritance concept") into two categories or parts :
  • उपवर्ग (subclass/child) - the class that inherits from another class
  • उच्चवर्ग(superclass/parent) - the class being inherited from
To inherit from a class, use the विरासेगा(extends) keyword.
In the example below, the "कार" वर्ग i.e. "कार" class (उपवर्ग/subclass) inherits the गुण(attributes) and विधियाँ(methods) from the "वाहन" वर्ग i.e. "वाहन" class (उच्चवर्ग/superclass): प्रोग्रामिंग में, गुणों और विधियों को एक वर्ग से दूसरे वर्ग में प्राप्त करना संभव है। हम "विरासत विषय"("inheritance concept") को दो श्रेणियों में समूहित करते हैं:
  • उपवर्ग/छोटा वर्ग(subclass/child) - वह वर्ग जो किसी अन्य वर्ग से विरासत में मिली है
  • उच्चवर्ग/बड़ा वर्ग (superclass/parent) - वर्ग जिस से विरासत मिली है
किसी वर्ग को विरासत मे पाने के लिए,विरासेगा(extends) शब्द का उपयोग करें।
नीचे दिए गए उदाहरण में, "कार" वर्ग (उपवर्ग), वाहन वर्ग (उच्चवर्ग) से गुण(attributes) और विधियाँ(methods) प्राप्त करता है: |("कार" class(subclass) inherits methods and attributes from "वाहन" class)

Example:

उदाहरण:
वर्ग वाहन { सुरक्षित डोर कंपनी = "Ford"; //वाहन गुण/attribute सार्वजनिक रिक्त हॉर्नबजाओ() { //वाहन विधि/method तंत्र.बाहर.प्रिंट("ब्रूम ब्रूम...." ); } } वर्ग कार विरासेगा वाहन { निजी डोर मॉडलनाम = "Mustang"; // Car attribute सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ //कार वर्ग की वस्तु बनाए कार मेरिकार =नई कार(); // मेरिकार वस्तु पर हॉर्नबजाओ विधि का इस्तेमाल करे (call kare) मेरिकार.हॉर्नबजाओ(); //कार की कंपनी का नाम और उसके मोडेल का नाम लिख दे तंत्र.बाहर.प्रिंट(मेरिकार.कंपनी +" "+मेरिकार.मॉडलनाम); } }

In the above example "कार विरासेगा वाहन"("कार inherits वाहन") means the "कार" वर्ग("कार" class) will inherit the methods and attributes of the "वाहन" वर्ग("वाहन" class). उपरोक्त उधारण मे "कार विरासेगा वाहन" का मतलब है कार वर्ग को वाहन वर्ग की विधियाँ(methords) और गुण(attributes) विरासत मे मिलेंगे|

Note :

Did you notice the सुरक्षित(protected) modifier in वाहन ?

We set the "कंपनी" गुण ("कंपनी" attribute) in वाहन to a सुरक्षित(protected) access modifier. If it was set to निजी(private), the "कार" वर्ग ("कार" class) would not be able to access it.

Why And When To Use "विरासत/Inheritance"?

- It is useful for code reusability: reuse गुण(attributes) and विधियाँ(methods) of an existing वर्ग(class) when you create a new वर्ग(class).
Tip: Also take a look at the next topic, which uses inherited methods to perform different tasks.

ध्यान दे:

क्या आपने "वाहन" में सुरक्षित संशोधक देखा ?

हमने "वाहन" में "कंपनी" विशेषता को सुरक्षित(protected) पहुच संशोधक पर सेट किया है। यदि इसे निजी(private) पर सेट किया गया होता, तो "कार" वर्ग इसका इस्तेमाल नहीं कर पाएगा।

"विरासत/inheritance" का उपयोग क्यों और कब करें?

- यह संहिता/कोड को पुन इस्तेमाल करने के लिए उपयोगी है: जब आप एक नया वर्ग(class) बनाते हैं तो किसी मौजूदा वर्ग(class) की विशेषताओं(attributes) और विधियों(methords) का पुन: उपयोग कर सकते हैं।



 The final Keyword| अंतिम शब्द

If we don't want other वर्ग(classes) to inherit from a वर्ग(class), use the अंतिम(final) keyword. यदि हम नहीं चाहते कि कोई अन्य वर्ग(class) किसी वर्ग(classes) को न विरासे , तो अंतिम(final) शब्द(कीवर्ड) का उपयोग करें।

Example:

If you try to inherit a final class you will get an error

यदि आप अंतिमवर्ग (final class) को विरसित करने का प्रयास करते हैं तो आपको एक त्रुटि मिलेगी
