 Abstract Classes and Methods| सार वर्ग और विधियाँ :

डेटा का सार निकालना(Data abstraction) is the process of hiding certain details and showing only essential information to the user. सारांश निकालना (Abstraction) can be achieved with either सार वर्ग or interfaces (abstract classes or interfaces) (which we will learn more about in the next chapter).

The सार(abstract) keyword is a गैर पहुच संशोधक (non-access modifier), used for वर्ग(classes) and विधियाँ(methods).
  • सारवर्ग (Abstract class): is a restricted वर्ग(class) that cannot be used to create वस्तु(objects) (to access it, it must be inherited from another वर्ग(class)).
  • सारविधि (Abstract method): can only be used in an सारवर्ग (Abstract class), and it does not have a body. The body is provided by the उपवर्ग (subclass inherited from).
An सारवर्ग (Abstract class) can have both सारविधि (Abstract methord) and regular methods(विधियाँ): डेटा का सार निकालना (Data abstraction) कुछ जानकारी छिपाने और उपयोग-कर्ता को केवल आवश्यक जानकारी दिखाने की प्रक्रिया है।

सारांश निकासी(Abstraction) ,सार वर्ग or interfaces (abstract classes or interfaces) से प्राप्त की जा सकता है। (जिसके बारे में हम अगले अध्याय में और जानेंगे)।

सार(सार) कीवर्ड एक गैर-पहुच संशोधक (गैर-पहुंच संशोधक) है, जिसका उपयोग वर्ग(वर्ग) के लिए किया जाता है। और विधियाँ(विधियाँ):

सारवर्ग(abstract class) : एक प्रतिबंधित वर्ग(class) है जिसका उपयोग वस्तु (objects) बनाने के लिए नहीं किया जा सकता (इस तक पहुँचने के लिए, इसे अवश्य ही अन्य वर्ग(class) से विरासत में मिला होना चाहिए।

सारविधि(Abstract method) : केवल एक सारवर्ग(Abstract class) में इस्तेमाल किया जा सकता है। और इसका कोई शरीर नहीं है। शरीर उपवर्ग(subclass) द्वारा प्रदान किया जाता है (उपवर्ग से विरासत में मिला)।

एक सारवर्ग(abstract class) में सारविधि (abstract methords) और आम/साधी विधियाँ(regular/normal methords) दोनों हो सकते हैं।

सार वर्ग जानवर { सार्वजनिक सार रिक्त जांवरकीआवाज़(); सार्वजनिक रिक्त नींद(){ तंत्र.बाहर.प्रिंट("zzz..."); } }


From the example above, it is not possible to create a वस्तु(object) of the "जानवर" वर्ग(class): ऊपर दिए गए उदाहरण से, "जानवर" वर्ग(class) का वस्तु(object) बनाना संभव नहीं है:

जानवर मेराजानवर = नई जानवर();

To access the सारवर्ग(Abstract class), it must be inherited from another वर्ग(class). Let's convert the "जानवर" वर्ग(class) we used in the last chapter to an सारवर्ग(Abstract class):(Remember from the previous chapter that we use the extends keyword to inherit from a class.) सारवर्ग (Abstract class) तक पहुँचने के लिए, इसे दूसरे वर्ग (class) से विरासीत किया जाना चाहिए। आइए "जानवर" वर्ग (class) को एक सारवर्ग (Abstract class) में परिवर्तित करें, जिसका हमने पिछले अध्याय में उपयोग किया था:(याद करे , पिछले अध्याय मे हम कक्षा/वर्ग(class) से विरासत (inherit) करने के लिए विरसेगा शब्द (कीवर्ड) का उपयोग करते हैं।)

Example:

उदाहरण:


//सार वर्ग बनाए create an abstract class सार वर्ग जानवर{ //सार विधि जिसमे कोड़(शरीर) नही है //Abstract method (does not have a body) सार्वजनिक सार रिक्त आवाज़करेगा(); //एक सामान्य विधि //a regular methord सार्वजनिक रिक्त सोजाओ(){ तंत्र.बाहर.प्रिंट(" जानवर सोया है ,Zzzz.... | "); } } //उपवर्ग बनाए create a subclass वर्ग कुत्ता विरासेगा जानवर { सार्वजनिक रिक्त आवाज़करेगा(){ //आवाज़करेगा() का कोड/शरीर यहाँ दिया जाएगा // The body of आवाज़करेगा() is provided here तंत्र.बाहर.प्रिंट(" कुत्ता भौंका , भों-भों.... | "); } } वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ कुत्ता मेराकुत्ता = नई कुत्ता(); मेराकुत्ता.आवाज़करेगा(); मेराकुत्ता.सोजाओ(); } }

In the above example if we do not provide a body(i.e. code) of "आवाज़करेगा()" विधि (methord) then we would be thrown an error as the compiler does not know what to do in that methord we must provide implementation of सार विधियाँ (abstract methords) .This process is also a case of methord overriding .(The concept is very(exactly) similar to overriding in java for any confusion(or depth knowledge) you can refer to java overriding concept available on internet) उपरोक्त उदाहरण में यदि हम "आवाज़करेगा ()" विधि (methord) का एक शरीर(अर्थात कोड) प्रदान नहीं करते हैं, तो हमें एक त्रुटि मिलेगी क्योंकि कम्पाइलर(या कहे कंप्युटर) को यह नहीं पता है कि उस विधि में क्या करना है, हमें उसका कार्यान्वयन(शरीर) प्रदान करना होगा सार विधियाँ (abstract methords)। यह प्रक्रिया भी methord overriding का मामला है।(methord overriding बहुत ही (बिल्कुल) जावा ओवरराइडिंग के समान है , किसी भी भ्रम (या गहराई तक जाने) के लिए आप इंटरनेट पर उपलब्ध जावा विधि ओवरराइडिंग (java methord overriding) पढ़ सकते है
