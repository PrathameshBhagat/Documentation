 Parameters and Arguments| विधियों के मापदंड/मानदंड और उनके तर्क(दलीले)

Information can be passed to methods(विधि) as parameter(मापदंड/मानदंड). Parameters(मापदंड/मानदंड) act as variables inside the method.
Parameters(मानदंड) are specified just after the method name, inside the parentheses (). We can add as many parameters(मानदंड) as we want, just separate them with a comma , .
The below example has a method that takes a डोर(String) called नाम as parameter(मानदंड). When a method is called, we pass along a first name, which is used inside the method to print the full name: जानकारी , विधियों को पैरामीटर (मापदंड/मानदंड) के रूप में प्रदान की जा सकती है। मापदंड/मानदंड विधि के अंदर वेरिएबल के रूप में कार्य करते हैं।
मानदंड विधि के नाम के ठीक बाद कोष्ठक () के अंदर लिखे जाते हैं। हम जितने चाहें उतने मानदंड(पैरामीटर) जोड़ सकते हैं, बस उन्हें "कॉमा" , से अलग करें।
नीचे दिए गए उदाहरण में एक विधि है जो एक डोर ,"नाम" को मानदंड(पैरामीटर) के रूप में लेती है| जब किसी विधि को बुलाया(कॉल किया) जाता है, तो हम पहले नाम प्रदान करते हैं, जिसका उपयोग विधि के अंदर पूरा नाम प्रिंट करने के लिए किया जाता है:

Example:

उदाहरण:
सार्वजनिक कक्षा Main{ स्थिर रिक्त मेरीविधि(डोर नाम){ तंत्र.बाहर.प्रिंट( नाम + " मनोहर | "); } सार्वजनिक स्थिर रिक्त जरूरी(डोर[] मान){ मेरीविधि("राम");//राम मनोहर | मेरीविधि("गीता");// गीता मनोहर | मेरीविधि("जय");//जय मनोहर | } }

Note :

When a मानदंड(parameter) is passed to the विधि (method), it is called a तर्क(argument). So, from the example above: नाम is a मानदंड(parameter), while राम , गीता and जय are तर्क(arguments).

ध्यान दे:

जब एक मानदंड(पैरामीटर) को विधि (पद्धति) में प्रदान/पास किया जाता है, तो इसे तर्क(argument) कहा जाता है। इसलिए, ऊपर दिए गए उदाहरण से: "नाम" मानदंड(पैरामीटर) है, जबकि राम, गीता और जय तर्क(arguments) हैं।



 Multiple Parameters|एकाधिक मानदंड

Example:

उदाहरण:
सार्वजनिक कक्षा Main{ स्थिर रिक्त मेरीविधि(डोर नाम, अंक उम्र ){ तंत्र.बाहर.प्रिंट( नाम + " मनोहर की उम्र " + उम्र +" है |\n"); } सार्वजनिक स्थिर रिक्त जरूरी(डोर[] मान){ मेरीविधि("राम",10);//राम मनोहर | मेरीविधि("गीता",15);// गीता मनोहर | मेरीविधि("जय",25);//जय मनोहर | //हम नई रेखा केलिए \n का प्रयोग कर सकते है //जैसे मेरीविधि मे किया गया है | } }

Note :

Note when we are working with multiple parameters(मानदंड), the विधि(method) call must have the same number of arguments(तर्क) as there are parameters(मानदंड), and the arguments(तर्क) must be passed in the same order.

ध्यान दे:

ध्यान दें जब हम कई मानदंड(पैरामीटर्स) के साथ काम कर रहे हों, तो विधि(मेथड) कॉल में मानदंड(parameters) के समान तर्कों(arguments) की संख्या होनी चाहिए, और तर्कों(arguments) को उसी क्रम में पारित किया जाना चाहिए।



 Return Values|वापसी मान

The रिक्त (void) keyword, used in the examples above, represents that the विधि(method) should not return a value. If we want the विधि(method) to return a value, we can use a आदिम (primitive) data type (such as अंक , बाइट , etc.) instead of रिक्त(void), and use the वापस (return) keyword inside the विधि(method): रिक्त(void) शब्द, ऊपर दिए गए उदाहरणों में इस्तेमाल किया गया है, यह दर्शाता है कि विधि(method) को मान(value) नहीं लौटाना चाहिए। यदि हम चाहते हैं कि विधि(method) एक मान लौटाए, तो हम रिक्त (void) के बजाय आदिम (primitive) डेटा प्रकार (जैसे अंक , बाइट, आदि) का उपयोग कर सकते हैं, और विधि के अंदर वापस (वापसी) शब्द (keyword) का उपयोग करना होगा :

Example:

उदाहरण:
सार्वजनिक कक्षा Main{ स्थिर अंक मेरीविधि( अंक अ ){ वापस 5+अ ; } सार्वजनिक स्थिर रिक्त जरूरी(डोर[] मान){ तंत्र.बाहर.प्रिंट(मेरीविधि(10)); } }//15 (5 + 10) का उत्पादन करेगा

 A Method with अगर..या|वापसी मान

It is common to use अगर..या , जब , केलिए and other statements inside methods.Following example uses अगर..या statements inside methods: विधियों के अनंदर अगर..या , जब , केलिए और अन्य कथनों का प्रयोग सामान्य है। विधि के अंदर अगर..या कथन:

Example:

उदाहरण:
सार्वजनिक कक्षा Main{ // एक उम्रजांच विधि बनाए एक उम्र नामक अंक वेरीअबल के साथ स्थिर रिक्त उम्रजांच( अंक उम्र ){ अगर (उम्र <18){ तंत्र.बाहर.प्रिंट("पहुंच अमान्य - माफ करे आप की उम्र पर्याप्त नहीं है "); } या { तंत्र.बाहर.प्रिंट("पहुंच मान्य - आपकी उम्र पर्याप्त है "); } } सार्वजनिक स्थिर रिक्त जरूरी(डोर[] मान){ उम्रजांच(19); } }//पहुंच मान्य - आपकी उम्र पर्याप्त है का उत्पादन करेगा
