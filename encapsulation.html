 Encapsulation | संपुटिकरण |सुरक्षण

The meaning of संपुटिकरण/सुरक्षण (Encapsulation)(here in our programming language), is to make sure that "sensitive" data is hidden from users. To achieve this, you must:
  • declare वर्ग वेरीअबल,गुण(विशेषता)(class variables/attributes) as निजी
  • provide सार्वजनिक प्राप्तकरे और बदलदे( सार्वजनिक(public) get and set) methods to access and update the value of a निजी(private) variable.
 संपुटिकरण/सुरक्षण (Encapsulation) का अर्थ(यहाँ हमारी प्रोग्रामिंग भाषा मे) यह सुनिश्चित करना है कि "संवेदनशील" डेटा उपयोग-कर्ताओं से छिपा हुआ रहे । इसे प्राप्त करने के लिए, आपको:
  • वर्ग वेरीअबल/गुण(विशेषता)(class variables/attributes) को निजी(private) घोषित करना होगा
  • प्राप्तकरे और बदलदे विधियों को सार्वजनिक के रूप में घोषित कर उन्हें निजी चर(private variable) तक पहुँचने(निर्धारित करने और बदलने) के लिए प्रदान करना होगा (हम इन्हे प्राप्त और बदले नाम देंगे)

 Get and Set|प्राप्त और बदले

We learned from the previous topic that निजी(private) variables can only be accessed within the same वर्ग(class) (i.e. an outside वर्ग(class) has no access to it). However, it is possible to access them if we provide सार्वजनिक प्राप्त और बदले विधियाँ (सार्वजनिक(public) get and set methods).

The प्राप्त(get) method returns the variable value, and the बदले(set) method sets and updates the value.
Syntax for both is that they start with either प्राप्त(get) or बदले(set), followed by the name of the variable: हमने पिछले विषय से सीखा है कि निजी चरों (private variables )को केवल उसी वर्ग(class) के भीतर ही पहुँचा जा सकता है (एक बाहरी वर्ग(class) की उस तक पहुँच नहीं है)। हालांकि, यदि हम सार्वजनिकप्राप्त और बदले विधियाँ (public get and set methods) का विकल्प प्रदान करते हैं, तो उन्हें इस्तेमाल(एक्सेस) करना संभव है।

प्राप्त(get) विधि चर मान(variable value) लौटाती है, और बदले(set) विधि मान को सेट और बदलती है।

Example:

उदाहरण:
वर्ग आदमी { निजी डोर नाम ; सार्वजनिक डोर नामप्राप्तकरे(){ वापस नाम ; } सार्वजनिक रिक्त नामबदलदे(डोर वर्तमाननाम){//या रख दे इस.नाम = वर्तमाननाम ; } }


The प्राप्तकरे(get) method returns the value of the variable name.

The बदलदे(set) method takes a मानदंड(parameter) "वर्तमाननाम" and assigns it to the "नाम" variable. The इस(this) keyword is used to refer to the current वस्तु(object).

However, as the "नाम" variable is declared as निजी(private), we cannot access it from outside इस वर्ग (this class) or we can say the object made from the present or current class: प्राप्तकरे(get) विधि चर(variable) "नाम" का मान(value) लौटाती है।

बदलदे(set) विधि एक मानदंड(parameter) "वर्तमाननाम" लेती है और इसे "नाम" वेरिएबल को प्रदान करती है|

इस(this) शब्द(कीवर्ड) का उपयोग वस्तु(object) को संदर्भित(इशारा) करने के लिए किया जाता है।
इस वर्ग (this class) या हम कह सकते हैं कि इस वर्ग से बनाई वस्तु से ताल्लुक़(connection) है:

Example:

उदाहरण:


वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ आदमी अ = नई आदमी(); अ.नाम="रोहन"; //यह त्रुटि देगा क्योंकि नाम निजी है आदमी वर्ग मे तंत्र.बाहर.प्रिंट(अ.नाम); } } वर्ग आदमी { निजी डोर नाम =" "; सार्वजनिक डोर नामप्राप्तकरे(){ वापस नाम ; } सार्वजनिक रिक्त नामबदलदे(डोर वर्तमाननाम){//या "रख" दे इस.नाम = वर्तमाननाम ; } }

If the variable was declared as सार्वजनिक(public), we would expect the output to be "रोहन" .

However, as we try to access a निजीprivate variable, we get an error.Instead, we use the "प्राप्तकरेनाम()"{getName()} and "बदलदेनाम()"{setName()} विधियाँ(methods) to access and update the variable: यदि चर(variable) को सार्वजनिक(public) के रूप में घोषित किया गया होता, तो हम उम्मीद करेंगे कि आउटपुट "रोहन" होगा |

हालांकि, जैसे हमने एक निजी चर (private variable) तक पहुँचने का प्रयास किया हैं, हमें एक त्रुटि मिली । इसके बजाय, हम चर(variable) तक पहुँचने और इस्तेमाल करने के लिए "प्राप्तकरेनाम()"{getName()} और "बदलेनाम()"{setName()} विधियों(methords) का उपयोग करते हैं तो हमे कोई तकलीफ नहीं होगी :

Example:

उदाहरण:
वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ आदमी अ = नई आदमी(); अ.नामबदलदे("रोहन"); तंत्र.बाहर.प्रिंट(अ.नामप्राप्तकरे()); //यह "रोहन" लिख देगा } } वर्ग आदमी { निजी डोर नाम =" "; सार्वजनिक डोर नामप्राप्तकरे(){ वापस नाम ; } सार्वजनिक रिक्त नामबदलदे(डोर वर्तमाननाम){//या "रख" दे इस.नाम = वर्तमाननाम ; } }




 Why Encapsulation?|सुरक्षण क्यों करे?

  • Control : To have better control of वर्ग गुण/विशेषताए,विधियाँ (class attributes and methods)
  • Limited Access: वर्ग गुण/विशेषताए(Class attributes/variables) can be made read-only (if you only use the प्राप्तकरे विधि (get method)), or write-only (if you only use the बदलदे विधि(set method))
  • Flexible: the programmer can change one part of the code without affecting other parts
  • Security: Increased security of data and variables
  • नियंत्रण :वर्ग गुण/विशेषताए,विधियाँ(class attributes and methods) पर बेहतर नियंत्रण के लिए
  • सीमित पहुच : वर्ग गुण/विशेषताए(class attributes) को केवल पढ़ने के लिए बनाया जा सकता है (यदि आप केवलप्राप्तकरे विधि(get method)का उपयोग करें), या केवल-लिखें (यदि आप केवल बदलदे विधि(set method) का उपयोग करते हैं)
  • लचीला: प्रोग्रामर अन्य भागों को प्रभावित किए बिना संहिता(code) के एक भाग को बदल सकता है
  • सुरक्षा: डेटा और वेरिएबल्स की बढ़ी हुई सुरक्षा
