  Exceptions| त्रुटियाँ/गलतीयाँ/चूक

While executing the code, different errors may occur: coding errors made by the programmer, errors due to wrong input, or other unforeseeable, unpredictable things.

When an error occurs, Compiler/Interpreter will normally stop and generate an error message. The technical term for this is: Java will throw an exception (throw an error). संहिता/कोड निष्पादित(execute) करते समय, विभिन्न त्रुटियां/गलतियां/चूक हो सकती हैं: प्रोग्रामर द्वारा की गई संहिता रचना मे त्रुटियां(coding mistakes/errors) , गलत इनपुट के कारण त्रुटियां, या अन्य अप्रत्याशित, अनजान चीजों के कारण त्रुटियां/गलतीयाँ हो सकती है ।

जब कोई त्रुटि होती य पाई जाती है , तो कंपाइलर/इंटरप्रेटर सामान्य रूप से रुक जाएगा और त्रुटि संदेश(error message) देता है । इसके लिए तकनीकी शब्द है: एक अपवाद (एक त्रुटि फेंक) फेंक देगा।



 Try......catch|कोशिश करो और पकड़े

The कोशिश(try) statement allows you to define a block of code to be tested for errors while it is being executed.
The पकड़े(catch) statement allows you to define a block of code to be executed, if an error occurs in the try block.
Both of them should come in pairs( else an error occurs) : कोशिश(try) कथन आपको संहिता/कोड के एक खंड को परिभाषित करने की अनुमति देता है, जसे त्रुटियों के लिए परीक्षण किया जाना है।

पकड़े(catch) कथन मे वो संहिता/कोड होता है जिसे तब चलाया जाना है जब प्रयास ब्लॉक में कोई त्रुटि पाई गई हो ।
दोनों को जोड़े में आना चाहिए (अन्यथा एक त्रुटि उतपन्न होती है):| हम कोशिश करते है की कोई त्रुटि/गलती/चूक ना हो यदि पाई जाती है तो हम उसे पकड़(पकड़े) लेते है |

(खंड => block i.e. code between { })

Syntax:

उदाहरण:
कोशिश { //संहिता/कोड जिसे चलाने की कोशिश की जानी है //block of try } पकड़े { //संहिता/कोड जिसे त्रुटि पाए जाने पर चलना है //block of try }

Consider the following example: निम्नलिखित उदाहरण देखे :

Example:

This will generate an error, because मेरीसंख्याएँ[10] does not exist.(it has only 4 numbers )

उदाहरण:

यह एक त्रुटि उत्पन्न करेगा, क्योंकि मेरीसंख्याएँ [10] मौजूद नहीं है। (इसमें केवल 4 हैं)
वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] args){ अंक [] मेरीसंख्याएँ = {1,2,3,4 };//ये एक त्रम है ,त्रम विषय मे पढे | तंत्र.बाहर.प्रिंट("अंक है : "+मेरीसंख्याएँ[10]);// त्रुटि/गलती/चूक !! } }

If an error occurs, we can use कोशिश....पकड़े(try...catch)to catch the error and execute some code to handle it: यदि कोई त्रुटि होती है या पाई जाती है , तो हम त्रुटि को पकड़ने के लिए कोशिश....पकड़े(try...catch) का उपयोग कर सकते हैं और इसे संभालने के लिए कुछ कोड निष्पादित(execute) कर सकते हैं:

Example:

उदाहरण:
वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ कोशिश { अंक [] मेरीसंख्याएँ = {1,2,3,4 };//ये एक त्रम है ,त्रम विषय मे पढे | तंत्र.बाहर.प्रिंट("अंक है : "+मेरीसंख्याएँ[10]);// त्रुटि/गलती/चूक !! } पकड़े (त्रुटि त ){ तंत्र.बाहर.प्रिंट("कुछ त्रुटि/गलती पाई गई है "); } } } //कुछ त्रुटि/गलती पाई गई है : लिख देगा



 Finally|अंतमे

The अंतमे(finally) statement lets you execute code, after कोशिश....पकड़े (try...catch), regardless of the result: अंतिम(finally) कथन आपको कोशिश....पकड़े(try...catch) के बाद कोड निष्पादित(execute) करने देता है, भले ही त्रुटि पाई जाए या ना पाई जाए :

Example:

उदाहरण:
वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ कोशिश { अंक [] मेरीसंख्याएँ = {1,2,3,4 };//ये एक त्रम है ,त्रम विषय मे पढे | तंत्र.बाहर.प्रिंट("अंक है : "+मेरीसंख्याएँ[10]);// त्रुटि/गलती/चूक !! } पकड़े (त्रुटि त ){ तंत्र.बाहर.प्रिंट("कुछ त्रुटि/गलती पाई गई है "); } अंतमे{ तंत्र.बाहर.प्रिंट(" कोशिश और पकड़े समाप्त हो गए है "); } } } //कुछ त्रुटि/गलती पाई गई है //कोशिश और पकड़े समाप्त हो गए है : लिख देगा



 throw keyword|फेके

The फेके(throw) statement allows you to create a custom error.

The फेके(throw) statement is used together with an त्रुटिexception type.

There are many त्रुटि(exception) types available : अंकगणितत्रुटि (ArithmeticException), फाइलप्राप्तीमेत्रुटि (FileNotFoundException), त्रमअनुक्रमांकसीमाकेबाहरत्रुटि (ArrayIndexOutOfBoundsException),सुरक्षात्रुटि (SecurityException), etc:

(in the foreseen example in line "पकड़े (त्रुटि त )" the word " त्रुटि " catches all त्रुटियां (exceptions) in one statment so you need not catch the exceptions one by one) फेके(throw) कथन आपको एक नया त्रुटि संदेश बनाने मे मदत करता है।
फेके(थ्रो) स्टेटमेंट का उपयोग त्रुटि के साथ किया जाता है अपवाद प्रकार।

कई त्रुटि(Exception) प्रकार उपलब्ध हैं: अंकगणितत्रुटि(ArithmeticException), फ़ाइलप्राप्तीमेत्रुति(FileNotFoundException), ट्रामअनुक्रमांकसीमाकेबाहारत्रुति (ArrayIndexOutOfBoundsException), सुरक्षात्रुटि (SecurityException), आदि: (पहले देखे उदाहरण मे इनलाइन "पकड़े (त्रुटि त)" में " त्रुटि " सभी त्रुटियों/अपवादों को एक विवरण में पकड़ता है, इसलिए आपको अपवादों को एक-एक करके पकड़ने की जरूरत नहीं है ना सभी त्रुटियों कॉ याद रखने की जरूरत है )

Example:

उदाहरण:
वर्ग मुख्य { स्थिर रिक्त उम्रजांच(अंक उम्र){ कोशिश { अगर (उम्र < 18 ) { फेके नई अंकगणितत्रुटि("प्रवेश निषिद्ध - उम्र कम से कम 18 होनी चाहिये | "); } या { तंत्र.बाहर.प्रिंट("प्रवेश मान्य - उम्र पर्याप्त है | "); } } पकड़े (त्रुटि त ){ तंत्र.बाहर.प्रिंट(त); } } सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ उम्रजांच(15);//यह एक त्रुटि देगा जिसमे संदेश होगा प्रवेश निषिद्ध.. .. .. होनी चाहिये | } } //अगर उम्र 22 होती तो "प्रवेश मान्य - उम्र पर्याप्त है |" : लिख देता

Note :

अंकगणितत्रुटि(ArithmeticException), फ़ाइलप्राप्तीमेत्रुति(FileNotFoundException), all these are just simple classes(वर्ग) and all of them have only one main उच्चवर्ग(superclass) - त्रुटि वर्ग(Exception class) i.e. all these (Arithm..) वर्ग(classes) are a उपवर्ग(subclass) of त्रुटि वर्ग(Exception class)

You can create your own त्रुटि(Exception) : simply create a वर्ग(class) and its superclass is to be त्रुटि वर्ग(Exception class) and अधिरोहे(override) its(त्रुटि वर्ग(Exception class))'s constructor in the constructor of your वर्ग(class).

In other words, to create a new त्रुटि(Exception), you have to create a वर्ग(class), then extend (त्रुटि वर्ग(Exception class)) in your वर्ग(class), then make a निर्मातक विधि(constructor) of your वर्ग(class) and in it call the निर्मातक विधि(constructor) of उच्चवर्ग(superclass) i.e. (त्रुटि वर्ग(Exception class)) with error message as मानदंड(parameter). (See the Example below :)

ध्यान दे:

अंकगणितत्रुटि(ArithmeticException), फ़ाइलप्राप्तीमेत्रुति(FileNotFoundException), ये सभी असल मे एक वर्ग(class) ही है और इन सब का एक ही मुख्य उच्चवर्ग(superclass) है - त्रुटि वर्ग(Exception class) और ये सभी(अंकगणि..) त्रुटि वर्ग के ही उपवर्ग(subclass) है |

आप आपनी खुद की त्रुटि बना सकते है : बस एक वर्ग बनाए और उसमे त्रुटि वर्ग को उच्चवर्ग बनाए फिर उस वर्ग की निर्मातक विधि बनाए और उसमे उच्चवर्ग की निर्मातक विधि को अधिरोहे (override) |


दूसरे शब्दों मे नई त्रुटि बनाने के लिए आपको एक वर्ग बनाना होगा , फिर उस वर्ग को आपने मे विरसना होगा त्रुटि वर्ग , फिर उसे एक निर्मातक विधि (constructor) बनाके उसमे उच्चवर्ग त्रुटि के निर्मातक विधि (constructor) को अपना त्रुटि संदेश देना होगा |(नीचे का उद्धरण देखे )

Example:

उदाहरण:
वर्ग मेरीगलती विरासेगा त्रुटि { सार्वजनिक मेरीगलती(डोर संदेश ) { उच्च(संदेश); } } वर्ग मुख्य { स्थिर रिक्त उम्रजांच(अंक उम्र){ कोशिश { अगर (उम्र < 18 ) { फेके नई मेरीगलती("प्रवेश निषिद्ध - उम्र कम से काम 18 होनी चाहिये | "); } या { तंत्र.बाहर.प्रिंट("प्रवेश मान्य - उम्र पर्याप्त है | "); } } // ये सभी त्रुटियाँ पकडलेगा पकड़े ( त्रुटि त ){ // ये सभी त्रुटियों के त्रुटि संदेश (error message) प्रिन्ट कर देगा तंत्र.बाहर.प्रिंट(त); // मेरीगलती को भी } } सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ उम्रजांच(15);//यह एक त्रुटि देगा जिसमे संदेश होगा प्रवेश निषिद्ध.. .. .. होनी चाहिये | } } //"मेरीगलती: प्रवेश निषिद्ध - उम्र कम से काम 18 होनी चाहिये |" : लिख देगा
