 Polymorphism | बहुरूपता:

बहुरूपता(Polymorphism) means "many forms", and it occurs when we have many वर्ग(classes) that are related to each other by विरासत(inheritance).

Like we specified in the previous chapter; विरासत(Inheritance) lets us inherit गुण(attributes) and विधियाँ(methods) from another वर्ग(class). बहुरूपता(Polymorphism) uses those विधियाँ(methods) to perform different tasks. This allows us to perform a single action in different ways.

For example , think of a उच्चवर्ग(superclass) called "जानवर" (i.e. Animal) that has a विधियाँ(methods) called जांवरकीआवाज़(). उपवर्ग(Subclasses) of "जानवर" (i.e. Animal) could be कुत्ता , बिल्ली ,चिड़िया ,सूअर (Pigs, Cats, Dogs, Birds)- And they also have their own implementation of an animal sound (the pig oinks, and the cat meows, etc.): बहुरूपता(Polymorphism) का अर्थ है "कई रूप", और यह तब होता है जब हमारे पास कई वर्ग(classes) होते हैं जो विरासत(inheritance) द्वारा एक दूसरे से संबंधित या जुड़े होते हैं।

जैसा कि हमने पिछले अध्याय में स्पष्ट किया था; विरासत(Inheritance) हमें दूसरे वर्ग(class) से गुण(attributes) और विधियाँ(methords) प्राप्त करने देती है। बहुरूपता(Polymorphism) विभिन्न कार्यों को करने के लिए उन विधियाँ(methords) का उपयोग करती है। यह हमें अलग-अलग तरीकों से एक ही क्रिया करने की अनुमति देती है।

उदाहरण के लिए, एक उच्चवर्ग (superclass) के बारे में सोचें जिसका नाम "जानवर" (i.e. Animal) है, उसमें जांवरकीआवाज() नामक विधियाँ(विधियाँ) हैं। "जानवर"(i.e. Animal) का उपवर्ग(subclass) कुत्ता , बिल्ली, चिड़िया, सूअर (Dogs, Cats, Birds, Pigs) - हो सकता है । और उनके पास एक पशु ध्वनि का अपना कार्यान्वयन(implementation) भी है (कुत्ता भौंकता है, और बिल्ली म्याऊ, आदि):

Example:

उदाहरण:
वर्ग जानवर{ सार्वजनिक रिक्त आवाज़करेगा(){ तंत्र.बाहर.प्रिंट("जानवर आवाज करता है"); } } वर्ग कुत्ता विरासेगा जानवर { सार्वजनिक रिक्त आवाज़करेगा(){ तंत्र.बाहर.प्रिंट("कुत्ता भौंकेगा "); } } वर्ग बिल्ली विरासेगा जानवर { सार्वजनिक रिक्त आवाज़करेगा(){ तंत्र.बाहर.प्रिंट("बिल्ली म्याऊँ -म्याऊँ करेगी "); } }


Now we can create "कुत्ता"(Dog) and बिल्ली (Cat) objects and call the आवाज़करेगा() method on both of them: अब हम "कुत्ता"(Dog) और बिल्ली (Cat) वस्तु बना सकते हैं और उन दोनों पर आवाज़करेगा() विधि का इस्तेमाल (कॉल) कर सकते हैं:

Example:

उदाहरण:
वर्ग जानवर{ सार्वजनिक रिक्त आवाज़करेगा(){ तंत्र.बाहर.प्रिंट(" जानवर आवाज करता है |"); } } वर्ग कुत्ता विरासेगा जानवर { सार्वजनिक रिक्त आवाज़करेगा(){ तंत्र.बाहर.प्रिंट(" कुत्ता भौंकेगा |"); } } वर्ग बिल्ली विरासेगा जानवर { सार्वजनिक रिक्त आवाज़करेगा(){ तंत्र.बाहर.प्रिंट(" बिल्ली म्याऊँ - म्याऊँ करेगी |"); } } वर्ग मुख्य { सार्वजनिक स्थिर रिक्त जरूरी(डोर[] क ){ जानवर मेराजानवर = नई जानवर(); कुत्ता मेराकुत्ता = नई कुत्ता(); बिल्ली मेरीबिल्ली = नई बिल्ली(); मेराजानवर.आवाज़करेगा(); मेराकुत्ता.आवाज़करेगा(); मेरीबिल्ली.आवाज़करेगा(); } }

In the above example a विधि(methord) with same name has different tasks to perform i.e. code implementation. This in here is called विधि विधि अधिरोहण(methord overriding) . Remember earlier in the "overloading/अतिभारे" topic we विधि अतिभारा था (overloaded methords) i.e. we had methords with same name but different मानदंड(parameters) or different number of मानदंड(parameters). उपरोक्त उदाहरण में एक ही नाम की विधियों(methord) में करने के लिए अलग-अलग कार्य हैं (कोड कार्यान्वयन/code implementation)| इसे यहां विधि अधिरोहण/विधि अवहेलना/विधि उल्लंघन (methord overriding) कहा जाता है। याद करे "overloading/अतिभारे" में हमने विधियों को अतिभारा था (overloaded methords), यानी हमारे पास एक ही नाम के विधियों(methords) थे लेकिन अलग-अलग मापदंड (parameter) थे (या अलग-अलग संख्या संख्या मे थे)।


Here(in अधिरोहण/overriding) we have methords( आवाज़करेगा() ) with exact
  • same name
  • same मानदंड/parameters (same type and number)
but different implementation of code in them , which code to run this is decided by which वस्तु(object) call's the methord .
यहाँ(विधि अधिरोहण(methord overriding) में) हमारे पास
  • समान नाम
  • समान मानदंडों/parameter(समान प्रकार और संख्या)
के साथ विधियाँ (methords) हैं, लेकिन उनमें विभिन्न संहिता/कोड कार्यान्वयन(code implementation) हैं, कौन सा संहिता/कोड चलेगा ये इससे तेय होगा कि किस वस्तु की विधि का इस्तेमाल किया गया है(methord is called by which object) ।



In simple words object would decide which code implementation is to be called if कुत्ता.आवाज़() the code provided by the "कुत्ता" वर्ग ("कुत्ता" class) would run if "बिल्ली" then implementation provided by "बिल्ली" and so on .
(The concept is very(exactly) similar to overriding in java for any confusion(or depth knowledge) you can refer to java overriding concept available on internet)

सरल शब्दों में वस्तु(object) तय करेगा कि कौन से संहिता/कोड को चलाया जाना है : यदि कुत्ता.आवाज़() तो "कुत्ता" वर्ग ("कुत्ता" class) द्वारा प्रदान किया गया संहिता/कोड चलाया जाएगा ,"बिल्ली" तो "बिल्ली" द्वारा प्रदान किया गया संहिता/कोड चलाया जाएगा और इसी तरह आगे।(यह बहुत ही (बिल्कुल) जावा(java programming language)के समान है , किसी भी भ्रम (या गहरा अध्ययन) के लिए , आप इंटरनेट पर उपलब्ध जावा विधि अधिरोहण(java methord overriding) अवधारणा को पढ़ सकते हैं|)




Note :

Why And When To Use "विरासत"("Inheritance") and "बहुरूपता"("Polymorphism")? - It is useful for code reusability: reuse attributes and methods of an existing class when you create a new class. Also to have shorter code .

ध्यान दे:

क्यों और कब "विरासत"("Inheritance") और "बहुरूपता"("Polymorphism") का उपयोग करना है?- यह कोड का पुन उपयोग करने के लिए जरूरी है: जब आप एक नई कक्षा बनाते हैं तो मौजूदा वर्ग की विशेषताओं और विधियों का पुन: उपयोग कर सकते है । साथ ही कोड छोटा भी होगा ।
