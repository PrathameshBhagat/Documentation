 Switch | बदले | बदलों:

In place of writing many अगर....याstatements, we can use the बदले statement.
The बदले statement selects one(or some) of many provided code blocks to be executed: कई अगर .... या लिखने के स्थान पर , हम बदले कथन का उपयोग कर सकते हैं।
बदले कथन निष्पादित(execute) किए जाने वाले कई दिए गए कोड ब्लॉकों में से एक (या कुछ) का चयन करता है:

Syntax:

वाक्यविन्यास:
बदले (expression) { यदि x: // code block भंग; यदि y: // code block भंग; साधारण: // code block }

The बदलेexpression is evaluated/calculated once.
The value of that expression is then compared with the values of each given case after यदि keyword .
If a match is found, the respective block of code is executed.
(The भंग and साधारण keywords are optional, and are to be described later)

The following example uses the week's day number to find the week's day name: बदलेअभिव्यक्ति का मूल्यांकन/गणना एक बार की जाती है।
उस अभिव्यक्ति के मान(value) की तुलना यदि के बाद दिए गए प्रत्येक मामले के मूल्यों से की जाती है।
यदि कोई मिलान पाया जाता है, तो कोड के संबंधित ब्लॉक को निष्पादित(execute) किया जाता है।
(विराम और साधारण वैकल्पिक हैं, और बाद में वर्णित किए जाने हैं)

निम्न उदाहरण सप्ताह के दिन का नाम ढूँढने के लिए सप्ताह के दिन संख्या का उपयोग करता है:

Example:

उदाहरण:
इंट day = 4; बदले (day) { यदि 1: तंत्र.बाहर.प्रिंट("Monday"); भंग; यदि 2: तंत्र.बाहर.प्रिंट("Tuesday"); भंग; यदि 3: तंत्र.बाहर.प्रिंट("Wednesday"); भंग; यदि 4: तंत्र.बाहर.प्रिंट("Thursday"); भंग; यदि 5: तंत्र.बाहर.प्रिंट("Friday"); भंग; यदि 6: तंत्र.बाहर.प्रिंट("Saturday"); भंग; यदि 7: तंत्र.बाहर.प्रिंट("Sunday"); भंग; } // Outputs "Thursday" (day 4)


 The भंग Keyword| भंग कथन

When system reaches a भंग keyword, it breaks out of the बदले block.
And this will stop the execution of more code and case testing given in the block.
If a match is found, and the job is done, it's time for a break or a भंग. No need for more testing. जब सिस्टम एक भंग तक पहुंचता है, तो यह बदले ब्लॉक से बाहर निकल जाता है।
और इससे ब्लॉक में दिए गए अधिक कोड और केस टेस्टिंग का निष्पादन(execution) बंद हो जाएगा।
यदि कोई मेल पाया जाता है, और काम पूरा हो जाता है, तो यह ब्रेक या भंगकरने का समय है। अधिक परीक्षण की आवश्यकता नहीं है।

Note :

A break (भंग) can save a lot of execution time as it "ignores" the execution of all the remaining code in the switch(बदले) block.

ध्यान दे:

एक ब्रेक (भंग) बहुत सारे निष्पादन को लगने वाले समय को बचा सकता है क्योंकि यह बदले ब्लॉक में शेष सभी कोड के निष्पादन को "अनदेखा" करता देता है।


 The साधरण(default) Keyword| साधरण कथन

साधरण specifies some code which is to run if there is no case match in the बदले block. साधरण कुछ कोड निर्दिष्ट(execute) करता है जो बदले ब्लॉक में कोई केस मैच नहीं होने पर चलाना जाना है ,या कहे जिसे साधरण तौर पर चलना है |

Example:

उदाहरण:
इंट day = 4; बदले (day) { यदि 6: तंत्र.बाहर.प्रिंट("Today is Saturday"); break; यदि 7: तंत्र.बाहर.प्रिंट("Today is Sunday"); break; साधारण: तंत्र.बाहर.प्रिंट("Looking forward to the Weekend"); } // Outputs "Looking forward to the Weekend"

Note :

Note that , if the default(साधरण) statement is used as last statement in a switch(बदले) block, it does not need a भंग.

ध्यान दे:

ध्यान दें कि, यदि साधरण कथन का उपयोग बदले ब्लॉक में अंतिम कथन के रूप में किया जाता है, तो इसे भंग की आवश्यकता नहीं है।
