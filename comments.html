  Comments| टिप्पणियाँ:

Comments can be used to explain code, and to make it more readable. It can also be used to prevent execution when testing alternative code. संहिता(कोड/code) को समझाने और इसे और अधिक पढ़ने लायक बनाने के लिए टिप्पणियों(कमेंट्स) का उपयोग किया जा सकता है। इसका उपयोग वैकल्पिक(ऑल्टर्नेट) संहिता(कोड/code) का परीक्षण करते समय निष्पादन को रोकने के लिए भी किया जा सकता है।

 Single-line Comments|एक पंक्ति टिप्पणियाँ|एक रेखा(लाइन) मे टिप्पणियाँ

Single-line comments start with two forward slashes //.
Any text between // and the end of the line is ignored by compiler (will not be executed). एक पंक्ति टिप्पणियाँ(एक रेखा/लाइन टिप्पणियाँ) दो फ़ॉरवर्ड स्लैश // से शुरू होती हैं।
// और पंक्ति(लाइन) के अंत के बीच के किसी भी हिस्से को कंपाइलर(या कहे कंप्युटर) द्वारा अनदेखा किया जाता है।(निष्पादित किया जाएगा/पढ़ा नहीं जाएगा)

Example:

This example uses a single-line comment before a line of code:

उदाहरण:

यह उदाहरण संहिता/कोड की एक पंक्ति से पहले एक पंक्ति(रेखा) की टिप्पणी का उपयोग करता है।


This example uses a single-line comment at the end of a line of code: यह उदाहरण संहिता/कोड की एक पंक्ति के अंत में एक पंक्ति(रेखा) की टिप्पणी का उपयोग करता है।

 Multi-line Comments|बहु पंक्ति टिप्पणियाँ

Multi-line comments start with /* and ends with */.
Any text between /* and */ will be ignored by compiler. बहु-पंक्ति/रेखा टिप्पणियाँ /* से शुरू होती हैं और */ पर समाप्त होती हैं।
/* और */ के बीच किसी भी पाठ को संकलक(कम्पाइलर या कहे कंप्युटर) द्वारा अनदेखा कर दिया जाएगा।

Note :

Single or multi-line comments?
It is up to you which you want to use. Normally, we use // for short comments, and /* */ for longer.

ध्यान दे:

एक या बहु-पंक्ति टिप्पणियाँ?
यह आप पर निर्भर है कि आप किसका उपयोग करना चाहते हैं। आम तौर पर, हम छोटी टिप्पणियों के लिए // का उपयोग करते हैं, और /* */ लंबे समय के लिए।
